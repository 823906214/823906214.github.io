基于git代码托管的CocoaPods私有库的搭建

CocoaPods是iOS最常用的第三方库管理工具，我们可以通过CocoaPods发布自己的私有库以供内部人员的使用。下面我以免费的代码托管网站Coding为例。
比如说我们要将TESTComment作为私有库，首先我们在coding注册一个账号，并建立一个项目。这时可以拿到那个项目的仓库地址https://git.coding.net/zheizi/cocoaPod.git，在终端中输入命令
pod repo add TESTCommon https://git.coding.net/zheizi/cocoaPod.git
将TESTCommon添加到本地repo, 添加成功后可以在~/.cocoapods/repos/目录下可以看到官方的specs:master和刚刚加入的specs:TESTCommon。进入到~/.cocoapods/repos/，将自己本地的库PBEnginerTool移到TESTCommon目录下。在终端中输入
git add -A && git commit -m"Release 0.01"
git tag -m "new version" '0.0.1'
git push --tags
git push origin master
将本地库推送到远端。
接下来填写自己的podspec文件。 

Pod::Spec.new do |s|
s.name         = 'TESTCommon'
s.version      = '0.0.1'
s.summary      = 'ymt Category files'
s.homepage     = 'https://git.coding.net/zheizi/cocoaPod.git'
s.license      = 'MIT'
s.authors      = {'name' => 'email'}
s.platform     = :ios, '7.0'
s.source       = {:git => "https://git.coding.net/zheizi/cocoaPod.git", :tag => "#{s.version}"}
s.source_files = "PBEnginerTool", "PBEnginerTool/**/*.{h,m}"
s.requires_arc = true
end

然后输入命令行验证文件有效性
pod lib lint
如果报错按错误提示进行相应的修改。通过验证后添加到我们第一步创建的私有Spec repo中去在终端输入
pod repo push TESTCommon TESTCommon.podspec
添加成功之后TESTCommon中会包含Category库的podspec信息, 可以前往~/.cocoapods/repos下的TESTCommon文件夹中查看, 同时git服务器中的远端也更新了。
到这一步，我们的私有库已经弄好了。在使用时只要在podfile文件开头引入自己的源source ‘https://git.coding.net/zheizi/cocoaPod.git
在引入自己的源后需要再手动映入一个公开源source ‘https://github.com/CocoaPods/Specs.git‘，不然其他第三方的公共库会提示找不到。

参考链接:https://segmentfault.com/a/1190000007947371

基于Gitlab的CI系统搭建

本系统是搭建在CentOS7系统下的。CentOS7安装过程中选择最小安装。

  1. 使用yum安装wget， yum -y install wget
  2. 安装完成后替换yum源，先备份yum源 cd /etc/yum.repos.d/ mv CentOS-Base.repo CentOS-Base.repo.backup 替换源 wget http://mirrors.163.com/.help/CentOS7-Base-163.repo mv CentOS7-Base-163.repo CentOS-Base.repo
  3. 配置http和ssh环境
    yum install curl openssh-server postfix cronie
